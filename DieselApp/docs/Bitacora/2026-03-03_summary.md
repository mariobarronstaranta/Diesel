## 3. Bug Crítico: Fechas con Desfase UTC → CST (−6h)

**Raíz del problema:** `new Date("2026-02-26")` interpreta el string ISO como UTC medianoche. Al convertir a CST (UTC−6) retrocede 6 horas, mostrando el día anterior.

**Componentes afectados y corrección aplicada:**

| Componente                            | Función                 | Corrección                                                           |
| ------------------------------------- | ----------------------- | -------------------------------------------------------------------- |
| `ReporteConsumos.tsx`                 | `formatearFecha`        | `new Date(fecha + "T00:00:00")`                                      |
| `ReporteConsumosDetalleModal.tsx`     | `formatearFechaDisplay` | `new Date(fecha + "T00:00:00")`                                      |
| `ReporteRendimientosDetalleModal.tsx` | `formatearFecha`        | Split directo: `[y,m,d] = fecha.split("-")` → `` `${d}/${m}/${y}` `` |
| `ReporteLecturas.tsx`                 | `formatearFecha` + sort | Split directo + `localeCompare` para orden                           |

**Solución preferida:** Split directo del string ISO elimina la conversión de zona horaria por completo:

```typescript
const [y, m, d] = fecha.split("-");
return `${d}/${m}/${y}`;
```

## 5. Búsqueda Predictiva en Combos de SalidasDiesel

**Requerimiento:** Los combos de Unidad y Operador en la pantalla de Salidas deben permitir búsqueda por texto mientras siguen funcionando como selectores.

**Decisión técnica:** Se usó `react-select` con `Controller` de `react-hook-form`. Se crearon componentes nuevos sin modificar los originales.

**Nuevos componentes:**

- `ComboUnidadesSearchable.tsx` — reemplaza `ComboUnidades` solo en SalidasDiesel
- `ComboOperadoresSearchable.tsx` — reemplaza `ComboOperadores` solo en SalidasDiesel

**Componentes originales no modificados:** `ComboUnidades.tsx`, `ComboOperadores.tsx`

**Dependencia agregada:** `react-select` v5

**Campos en `SalidasDiesel.tsx`:**

- El `useForm` ahora expone `control` además de `register`
- Los combos searchable reciben `control` + `name` en lugar de `register`
- El valor guardado en BD no cambia (`IDUnidad`, `IDPersonal`)

## Lecciones Aprendidas

1. **`new Date(isoString)` sin hora es UTC** — Siempre agregar `T00:00:00` o usar split directo cuando la fecha viene de BD como string ISO.
2. **Vite no recarga `.env` en hot-reload** — Requiere reinicio completo del servidor.
3. **Patrón para combos searchable con RHF**: usar `Controller` + `react-select`, no `register`. El valor ID se mantiene; solo cambia la UI.
4. **No tocar componentes existentes** — Crear variantes `*Searchable` independientes preserva retrocompatibilidad.
